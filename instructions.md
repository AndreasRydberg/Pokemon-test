# Introduction

Welcome to the Pokemon Coding Challenge! In this challenge, you will be working with a dataset of Pokemon, each represented as a JSON object with various properties.

## Problem

Develop a Pokémon battle simulator where users can simulate battles between two teams of Pokémon. The simulator should take into account various properties in the dataset to determine the outcome of the battle.

## Dataset

The dataset can be downloaded from [here.](https://raw.githubusercontent.com/Biuni/PokemonGO-Pokedex/master/pokedex.json?_sm_au_=iVV5F7R6VsVpqJM6pGsWvKttvN1NG)

## Tasks

- Create an API endpoint that simulates a Pokemon battle between two teams of Pokemon
- Use factors such as type effectiveness, stats (weight, height, multipliers), etc, to determine the winning team.
- Should return a detailed battle log showing the flow of the battle.
- The battle logic in this simulator does not need to reflect the outcomes or mechanics from the official Pokémon games. Feel free to create your own system for determining the winner based on the dataset and relevant factors.

## Requirements

- Your application should be written in Node.js.
- Should be written in Typescript.
- Your code should be organized and easy to read.
- You should handle any potential errors that could occur.
- Dockerize application
- Load the dataset into a NoSQL database

## Submission

The expected time frame for this home assignment is considered to be around 5 hours, please document in the README if there were time constraints limiting your work and what you would've done if you'd had more time.

Please submit your completed application by providing a link to a Git repository containing your code. Include any necessary instructions for running your application in a README file.

_Good luck!_
